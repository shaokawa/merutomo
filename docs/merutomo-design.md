# MERUTOMO 設計書

プロジェクト「**メル友（MERUTOMO）**」の完成版設計ドキュメント一式をまとめたものです。

---

## Ⅰ. プロジェクト総括レポート：MERUTOMO

本プロジェクトは、タイパ至上主義の現代に対するアンチテーゼとして、**「一通のメールを待つ高揚感」**を再定義するマッチングプラットフォームです。

- **ビジョン:** 「届かない時間を、楽しむ。」
- **コアバリュー:** 匿名メールアドレスによるクローズドな対話、意図的な通信ラグ、そしてテキスト主体の深いコミュニケーション。
- **技術的アプローチ:**
  - **フロントエンド:** Next.jsによる爆速な操作感と、90年代を彷彿とさせる角張ったレトロUIの融合。
  - **バックエンド:** Go言語（Gin）による堅牢なAPI基盤と、AWS Lambdaによるコスト効率の高いメールリレー。
  - **メディア管理:** Cloudinaryによる画像の自動レトロ加工。

---

## Ⅱ. 要件定義書 (Requirements Definition)

### 1. 機能要件

- **レトロ掲示板:** ユーザーが自身の匿名アドレスと共に、自己紹介と「エモい」写真を投稿できる。
- **匿名メールリレーシステム:**
  - 独自ドメインのアドレス（`id@merutomo.jp`）を全ユーザーに付与。
  - 受信メールを解析し、実メアドへ転送。送信元を隠蔽しプライバシーを保護。
- **タイムディレイ・エンジン:** 送信から受信まで、30〜60分のランダムな遅延を発生させる。
- **検閲ロジック:** 本文内のSNS/LINE ID、電話番号等の自動検知と伏せ字化（`****`）。
- **画像加工エンジン:** アップロードされた画像を、Cloudinary側で自動的に低解像度・セピア・グレースケール等のレトロ調へ変換。

### 2. 非機能要件

- **コスト最適化:** AWS無料枠を極限まで活用し、月間3,000通までの運用コストを最小化。
- **セキュリティ:** メールヘッダーから送信元のIPアドレスや個人情報を削除する匿名化処理の徹底。
- **パフォーマンス:** 掲示板の表示はNext.jsのISR/SSRを活用し、ミリ秒単位でレンダリング。

## Ⅲ. 基本設計・仕様書 (Technical Specifications)

### 1. 技術スタック（完成版）

- **Frontend:** Next.js (App Router), Tailwind CSS (RetroUI/Neobrutalism系)
- **Backend API:** **Go (Gin Framework)**
- **Email Relay:** **Go (AWS Lambda)**, Amazon SES, Amazon S3
- **Database:** PostgreSQL (Supabase / Managed Service)
- **Image Cloud:** **Cloudinary** (Dynamic Transformation)
- **Queueing:** Amazon SQS (Delay queues)

### 2. システムアーキテクチャ

1. **投稿時:** Next.js → Cloudinary (画像UP) → Gin API (DB保存)。
2. **メール受信時:** SES (MX Record) → S3保存 → Lambda (Go) 起動。
3. **転送予約:** LambdaがSQSに「Message + Delay(30-60min)」をパブリッシュ。
4. **最終送信:** SQSトリガーでLambdaが再起動 → SES (Send API) でユーザーの実メアドへ。

### 3. データベース設計 (主要テーブル)

| テーブル名 | 主なカラム | 役割 |
| :--- | :--- | :--- |
| `users` | `id`, `real_email`, `proxy_email`, `plan_id` | ユーザー基本情報と転送設定。 |
| `posts` | `id`, `user_id`, `content`, `image_url` | 掲示板への投稿内容。 |
| `stamps` | `id`, `name`, `asset_url`, `price` | アイテム課金用のデジタル切手/便箋。 |

### 4. 収益化モデル

- **Free:** 1日3通まで送信可、ラグ60分、写真1枚。
- **Premium (月額制):** 送信無制限、ラグ15分、限定便箋の開放。
- **Item:** 特別な日のための「速達チケット」や「デコレーション切手」の販売。

## Ⅳ. 運用コスト試算 (無料枠終了後の目安)

1通（受信＋転送）の基本コストを約$0.03$円と仮定。

- **月間1,000通のリレー:**
  - SES (送受信): $0.20 USD
  - S3/Lambda/SQS: $0.10 USD未満（無料枠内）
  - **合計:** **約50円 / 月**
- **画像配信 (Cloudinary):** 無料枠内で月間数千枚の配信が可能。
