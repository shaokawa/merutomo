## Go初期化手順

最初のゴールは、`backend` で `Gin` が立ち上がり、`GET /health` が `200 OK` を返すところまでです。そこまでできれば、次に認証やDB接続を積めます。

**手順**
1. `backend` に入る
```bash
cd /Users/shiokawataiga/Desktop/go/merutomo/backend
```

2. Goモジュールを初期化する  
モジュール名は、あとでGitHubに載せるならリポジトリ準拠にしておくのが無難です。
```bash
go mod init github.com/あなたのGitHub名/merutomo/backend
```
ローカル確認だけなら仮で
```bash
go mod init merutomo/backend
```
でも進められます。

3. Ginを入れる
```bash
go get github.com/gin-gonic/gin
```

4. 最低限のディレクトリを作る
```bash
mkdir -p cmd/api internal/router
```

5. `cmd/api/main.go` を作る  
中身はこれで十分です。
```go
package main

import (
	"log"

	"github.com/gin-gonic/gin"
	"merutomo/backend/internal/router"
)

func main() {
	r := gin.Default()
	router.Setup(r)

	log.Println("server started on :8080")
	if err := r.Run(":8080"); err != nil {
		log.Fatal(err)
	}
}
```

6. `internal/router/router.go` を作る
```go
package router

import "github.com/gin-gonic/gin"

func Setup(r *gin.Engine) {
	r.GET("/health", func(c *gin.Context) {
		c.JSON(200, gin.H{
			"status": "ok",
		})
	})
}
```

7. import path を自分の `go mod init` に合わせて直す  
もし `go mod init github.com/xxx/merutomo/backend` にしたなら、`main.go` のこの行:
```go
"merutomo/backend/internal/router"
```
を
```go
"github.com/xxx/merutomo/backend/internal/router"
```
に変えてください。

8. 起動する
```bash
go run ./cmd/api
```

9. 動作確認する
別ターミナルで:
```bash
curl http://localhost:8080/health
```
期待する返り値:
```json
{"status":"ok"}
```

**ここで作られる構成**
```text
backend/
  go.mod
  go.sum
  cmd/
    api/
      main.go
  internal/
    router/
      router.go
```

**補足**
- `cmd/api` はアプリの起動入口です。
- `internal` は外部から直接使わせたくないアプリ内部実装を置く場所です。
- 今は `router` だけ分ければ十分です。認証、DB、設定は次に切ります。

ここまで自分でやってみてください。終わったら次に、`config` と `.env` を入れて「環境変数を読むAPIサーバ」に進めます。
