## 2026-04-01 public.users 導入と migration 整備

### 今日やったこと

- `public.users` を導入
  - `auth.users` と 1:1 で紐づくアプリ側プロフィールテーブルを設計
  - カラムは `display_name`, `username`, `proxy_email` を採用
  - `username` は英小文字を許容し、`[a-z0-9_]`, 3-30 文字に制限

- 認証APIをプロフィール対応に変更
  - `POST /auth/register` が `email`, `password` に加えて `display_name`, `username` を受け取るように変更
  - Supabase Auth で `auth.users` 作成後、`public.users` に 1 行作成する実装を追加
  - `proxy_email` は `username@merutomo.jp` を backend 側で生成するようにした

- `login` / `me` のレスポンスを拡張
  - `POST /auth/login` と `GET /auth/me` が `display_name`, `username`, `proxy_email` を返すようにした
  - Welcome page で表示名を出せる backend の前提を整えた

- `public.users` 用のストアを追加
  - `CreateProfile`, `FindProfileByID` を持つ PostgreSQL ストアを追加
  - `username` 重複時のハンドリングを追加

- migration 運用を整備
  - 手動 SQL ではなく backend から流せるように `goose` を導入
  - `cmd/migrate` を追加し、`go run ./cmd/migrate up` で migration を実行できるようにした
  - `backend/migrations/00001_create_public_users.sql` を追加

- 動作確認
  - `go test ./...` は通過
  - `go run ./cmd/migrate status` で migration 状態を確認
  - Supabase ダッシュボード上で user 登録、login 成功まで確認

- Next.js frontend を作成
  - `frontend` 配下に Next.js App Router プロジェクトを作成
  - `register`, `login`, `welcome` の各画面を追加
  - `BACKEND_URL` を使う同一オリジン proxy route を `app/api/auth/*` に追加

- 認証フローを frontend から接続
  - `POST /api/auth/register` でユーザー登録できるようにした
  - `POST /api/auth/login` 成功時に access token を `localStorage` に保存するようにした
  - `GET /api/auth/me` を経由して backend の `/auth/me` を呼べるようにした
  - `welcome` page で `display_name` を表示するところまで実装した

- frontend の見た目を調整
  - テンプレート初期画面を置き換え、認証導線に合わせたレイアウトへ変更
  - フォント、配色、背景、ボタンスタイルを MERUTOMO 用に調整

- frontend の検証
  - `npm run lint` が通過
  - `npm run build` が通過

- 連絡先公開の設計を見直し
  - `proxy_email` 前提をやめ、`email_visibility` を `public`, `approval_required`, `private` の3段階で持つ方針に変更
  - `public.users` に `profile_text`, `avatar_url`, `email_visibility` を追加する migration を作成
  - `email_access_requests` テーブルを追加する migration を作成
  - backend の register/login/me レスポンスを `email_visibility` ベースに更新
  - frontend の register 画面で `email_visibility` を選べるようにし、welcome page でも表示するようにした

- UI導線を再構成
  - `/` をログイン後に戻るメイン画面へ変更
  - `welcome` は廃止し、アクセス時は `/` にリダイレクトするようにした
  - `/{username}` のページを追加し、右上の三点リーダーからログアウトできるようにした
  - ログアウト時は backend の `/auth/logout` を呼び、token を破棄して `/` に戻るようにした
  - `/` に固定の `+` 投稿ボタンを追加し、モーダルで投稿フォームを開くようにした
  - 未ログインで `+` を押した場合は login を促すメッセージを表示し、login ページへ誘導するようにした

- 投稿モーダルの見た目を調整
  - 投稿フォームを軽い composer 風のモーダルに変更
  - ヘッダー、本文入力、ツールバー、下部アクションを分けて調整
  - 画像入力は URL ではなくファイル選択に変更
  - メールアドレス欄と公開範囲の選択欄を追加
  - ログイン中ユーザーが投稿モーダルを開いたとき、登録済みメールアドレスと `email_visibility` が初期値で入るようにした

### ハマった点

- Supabase direct DB 接続
  - `DATABASE_URL` を direct DB に向けていたため、IPv6 経由で `no route to host` になった
  - pooler 接続に切り替えて解消した

- `public.users` 未作成時の login 失敗
  - `auth.users` への登録は成功していても、`public.users` が無いと login 後のプロフィール取得で失敗した
  - 以後は migration を先に流す運用に切り替える

- `profile_text` / `avatar_url` の nullable カラム
  - string にそのまま scan すると登録直後に失敗するので、backend 側で `coalesce` して読むようにした

### 今日の到達点

- frontend で register / login / me / welcome の最小フローが動く状態になった
- welcome page にログイン中ユーザーの `display_name` を表示できる状態になった
- 連絡先公開方針を `email_visibility` ベースに切り替える土台ができた
- `/` と `/{username}` を軸にした認証後UIの骨組みができた
